Scheduled Jobs: Page Layout Approach

How should the cron dashboard be structured? Pick the layout that fits your mental model.

A

Card Grid (like Bots page)

Each cron job gets a card with status indicator, next/last run, sparkline of recent runs, cost, and error count. Click a card to see full run history. Consistent with the existing Bots dashboard pattern.

Berlin Rental Tracker OK
0 5 * * * · scout · Every day 5am
Last: 3h ago Next: 21h $0.04/run
●●●●●●● ●
Evening Reflection 2 errors
0 21 * * * · main · Every day 9pm
Last: 12h ago Next: 12h $0.12/run
●●● ●● ●●

Pros

  • Consistent with Bots page pattern
  • Great at-a-glance health overview
  • Rich per-job info without clicking

Cons

  • Takes more space (60 cards = lots of scrolling)
  • Need to click through for run history
B

Timeline Table (like Sessions page)

Sortable table where each row is a cron job with columns for status, schedule, last run, next run, avg cost, error rate, and a mini run-history bar. Click a row to expand or navigate to detail.

Job Agent Schedule Last Run Next Avg Cost Status Last 7 runs
Berlin Rental Tracker scout 0 5 * * * 3h ago 21h $0.04 OK ●●●●●●●
Evening Reflection main 0 21 * * * 12h ago 12h $0.12 ERR ●●●●●●●

Pros

  • Dense info — see all 60 jobs at once
  • Sortable (by error rate, cost, next run)
  • Consistent with Sessions page

Cons

  • Less visual impact than cards
  • Harder to spot failing jobs at a glance
C

Hybrid: Summary Strip + Table (Recommended)

Top section: 4 summary cards (active jobs, currently failing, next run countdown, daily cost). Below: the sortable table from option B. Glanceable health + dense data. Click any row to drill into run history with full session replay link.
